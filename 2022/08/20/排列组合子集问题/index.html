排列组合子集问题

元素无重不可重复选

力扣78求nums的子集

通过保证元素之间的相对顺序防止出现重复的子集

力扣77 求[1,n]所有可能的k个数的组合

​ 只需修改basecase 控制算法仅仅收集第k层节点的值

力扣46 返回可能的全排列

​ 之前通过start位置控制不出现start之前的元素，排列问题本身就是穷举元素的位置，所以不需要start，需要一个used数组标记哪些元素可以被选择

元素可重不可重复选

力扣90题给一个nums数组，里面可能包含重复元素，返回可能的子集

体现在图上是需要剪枝，减去相同的

力扣40 可能存在重复元素，请你计算 candidates 中所有和为 target 的子集

​ 输入 candidates 和一个目标和 target，从 candidates 中找出中所有和为 target 的组合，candidates 可能存在重复元素

​ 说这是一个组合问题，其实换个问法就变成子集问题了：请你计算 candidates 中所有和为 target 的子集

力扣47给你输入一个可包含重复数字的序列 nums，请你写一个算法，返回所有可能的全排列

​ 标准全排列算法之所以出现重复，是因为把相同元素形成的排列序列视为不同的序列，但实际上它们应该是相同的；而如果固定相同元素形成的序列顺序，当然就避免了重复。

​ 当出现重复元素时，比如输入 nums = [1,2,2’,2’’]，2’ 只有在 2 已经被使用的情况下才会被选择，同理，2’’ 只有在 2’ 已经被使用的情况下才会被选择，这就保证了相同元素在排列中的相对位置保证固定。

元素无重可重复选

力扣39，无重复元素的整数数组 candidates，找出 candidates 中可以使数字和为目标数 target 的所有组合。candidates 中的每个数字可以无限制重复被选取。

​ 之前的不可重复选其实用的时start来控制，要想每个元素可以重复选，就在进行下一层递归时，不改变start的值即可

元素可重可重复选

可重可重复选其实和不可重可重复选是一样的

总结

  1. 组合和子集其实是同一个问题，在回溯法中i从start开始

  2. 排列问题，回溯法i从0开始且需要一个used数组来标记是否使过某个元素

  3. 当元素可重时，需要先排序数组，再到for里面加一个去重的条件

  4. 当元素可重复选时只需控制start不变即可
